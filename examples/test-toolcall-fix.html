Test Scenario Overview

This page tests the critical fix for toolCall text persistence. The main issue was that toolCalls WITH text were disappearing when new history entries arrived.

Root Cause: The display decision was based on existingMessage.content which could be empty after a poll, instead of checking the cache.

Fix Applied: Now checks toolCallTextCache when deciding whether to keep a toolCall visible.

Manual Testing

Open the chat widget and observe the console for detailed logging:

  • [CACHE UPDATE] - New text stored in cache
  • [CACHE HIT] - Using cached text
  • [DISPLAY FIX] - Cache used for display decision
  • [KEEPING] - ToolCall kept visible
  • [REMOVING] - Element being removed (should NOT happen for toolCalls with text!)
